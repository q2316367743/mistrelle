## 实现讨论组「群聊模式」(Group Chat)

在 `mistrelle` 插件中为讨论组新增群聊模式（PageDiscussion 的「聊天」Tab，之前是空桩）。

### 核心设计
- 数据落地 `discussion/${id}/chat.json`（消息 + 上下文 + 过期小时数），`discussion/${id}/memory/YYYY-MM-DD.json` 按天记忆；写入经 `enqueueByPath` 串行化（复用 DiscussionRecordService 的路径命令队列）。
- 群聊引擎 `GroupChatEngine` 对标已有的 `DiscussionEngine`：每个被 `@` 的成员并发跑一个 `ToolChat`，注入「角色工具 + 记忆工具 + enableSkill」。
- **身份隔离**：transcript 给每条历史打 `[用户]/[a]/[b]/[摘要]` 标签；每个 AI 的 system prompt 声明「你是 X，其他成员是独立个体，不假借他人观点」。发给 AI 时**保留 @ 原文**，让 AI 理解分工。
- **上下文**：只注入当前 active 上下文；按 `lastActivityAt + 过期小时` 滚动过期；右上角 ContextBar 显示条数/剩余时间，支持「新上下文」「压缩（LLM 摘要后开新上下文）」+ 过期小时数设置。
- **无 @ 仅记录不回复**（用户确认）。
- **成员管理**：成员即 `AiDiscussionRole`，复用 `discussion-func.tsx` 的 `openDiscussionPut(id, onSaved)`（已加 onSaved 回调）增删改角色/提示词/工具。
- **@ 输入**：扩展 `useChatSender` 支持 `role` 标签（data-type=role/data-id），新建 `GroupChatSender.vue`。
- **记忆工具**：`buildMemoryTools(id)` 提供 `read_memory` / `write_memory`，每个成员 agent 内置。

### 关键文件
- 新建：`entity/ai/AiGroupChat.ts`、`modules/discussion/GroupChatEngine.ts`、`pages/discussion/chat/useGroupChat.ts` + `components/{GroupChatSender,GroupChatContextBar,GroupChatMembers,GroupChatMessageList}.vue`
- 修改：`DiscussionChatService.ts`(覆盖空文件)、`DiscussionRecordService.ts`(导出 buildDiscussionFolderPath + enqueueByPath)、`hooks/useChatSender.ts`(role 提及)、`pages/discussion/chat/DiscussionChat.vue`(完整实现)、`entity/ai/index.ts`、`pages/app/discussion-func.tsx`(onSaved)

## 修复 mentionSuggestion.ts 两个运行时崩溃（Tiptap v3 suggestion 陷阱）
用户运行报错：① `onKeyDown` 读 `props.items.length` → undefined；② `insertContentAt` "Unknown node type: undefined"。

根因与修复：
1. **onKeyDown 的 props 不含 items**：Tiptap 的 `SuggestionKeyDownProps`（onKeyDown 收到的）只有 `event/range/command/clientRect`，**没有 items**；且我之前自造的 `selectedIndex` 字段框架根本不回传。改为把 `items` / `selectedIndex` / `command` 提升为 `makeSuggestionRenderer` 返回的工厂闭包状态：onStart/onUpdate 写入，`onKeyDown` 读闭包。
2. **insertContentAt 数组里的裸字符串 `' '` 非法**：`[{type:'skillMention',...}, ' ']` 中 `' '` 不是合法 JSONContent，ProseMirror 对字符串取 `value.type`=undefined → "Unknown node type: undefined"。改为 `{ type: 'text', text: ' ' }`（skillMention 本身已正确注册，故首个元素没报错，证实节点名无误）。

经验：Tiptap suggestion 的 `onKeyDown` 不回传 `items`，需要的选择状态必须自建闭包；`insertContentAt(range, [...])` 数组元素必须是带 `type` 的 JSONContent，连一个空格也要写成 `{type:'text',text:' '}`。
