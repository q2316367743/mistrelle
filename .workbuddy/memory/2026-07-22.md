# 2026-07-22

## 实现讨论组「群聊模式」(Group Chat)

在 `mistrelle` 插件中为讨论组新增群聊模式（PageDiscussion 的「聊天」Tab，之前是空桩）。

### 核心设计
- 数据落地 `discussion/${id}/chat.json`（消息 + 上下文 + 过期小时数），`discussion/${id}/memory/YYYY-MM-DD.json` 按天记忆；写入经 `enqueueByPath` 串行化（复用 DiscussionRecordService 的路径命令队列）。
- 群聊引擎 `GroupChatEngine` 对标已有的 `DiscussionEngine`：每个被 `@` 的成员并发跑一个 `ToolChat`，注入「角色工具 + 记忆工具 + enableSkill」。
- **身份隔离**：transcript 给每条历史打 `[用户]/[a]/[b]/[摘要]` 标签；每个 AI 的 system prompt 声明「你是 X，其他成员是独立个体，不假借他人观点」。发给 AI 时**保留 @ 原文**，让 AI 理解分工。
- **上下文**：只注入当前 active 上下文；按 `lastActivityAt + 过期小时` 滚动过期；右上角 ContextBar 显示条数/剩余时间，支持「新上下文」「压缩（LLM 摘要后开新上下文）」+ 过期小时数设置。
- **无 @ 仅记录不回复**（用户确认）。
- **成员管理**：成员即 `AiDiscussionRole`，复用 `discussion-func.tsx` 的 `openDiscussionPut(id, onSaved)`（已加 onSaved 回调）增删改角色/提示词/工具。
- **@ 输入**：扩展 `useChatSender` 支持 `role` 标签（data-type=role/data-id），新建 `GroupChatSender.vue`。
- **记忆工具**：`buildMemoryTools(id)` 提供 `read_memory` / `write_memory`，每个成员 agent 内置。

### 关键文件
- 新建：`entity/ai/AiGroupChat.ts`、`modules/discussion/GroupChatEngine.ts`、`pages/discussion/chat/useGroupChat.ts` + `components/{GroupChatSender,GroupChatContextBar,GroupChatMembers,GroupChatMessageList}.vue`
- 修改：`DiscussionChatService.ts`(覆盖空文件)、`DiscussionRecordService.ts`(导出 buildDiscussionFolderPath + enqueueByPath)、`hooks/useChatSender.ts`(role 提及)、`pages/discussion/chat/DiscussionChat.vue`(完整实现)、`entity/ai/index.ts`、`pages/app/discussion-func.tsx`(onSaved)

### 验证
`pnpm check`(vue-tsc) + `pnpm exec eslint` + `pnpm build` 均通过。运行时自测未做（需桌面环境 utools）。

### 注意 / 后续
- AI 消息目前只持久化最终文本（toolcall/thinking 不落盘）；如需在消息流展示工具调用，需改为持久化 AIMessage content 数组。
- compressContext 用讨论组首个成员模型或默认助手模型做摘要，模型不可用则降级为直接开新上下文。

## 群聊 UI 交互迭代（微信风格）
按用户反馈调整三处交互：
1. **@ 选择改用 Popup**：`GroupChatSender.vue` 不再内联渲染建议列表，改为把输入框作为 `t-popup` 触发锚点、建议列表放入 `#content`（受控 `:visible="suggestions.length>0"`，placement=top-left）。注意 `t-popup` 的 `trigger` 不支持 `"manual"`，用受控 visible 即可。
2. **上下文信息 → 圆环 + 点击弹面板**：`GroupChatContextBar.vue` 改为 `t-progress type="circle"`（显示剩余时间百分比，过期变红）作为触发点，点击 `t-popup`(placement=bottom-right) 展开操作面板（消息数/状态/过期小时数/新上下文/压缩）。
3. **成员管理收进右上角「更多」→ 布局侧边栏（非 t-drawer）**：`DiscussionChat.vue` 用 `t-layout`/`t-content`/`t-aside` 实现（t-layout 含 t-aside 时自动横向 push）。`t-aside` 常驻渲染，展开/收起由 `width` 属性控制（320px / 0px），带 `width` 过渡；收起态去掉左边框与 padding、overflow 隐藏。`GroupChatMembers` 内嵌其中，「管理」按钮仍走 `openDiscussionPut`。
验证：`pnpm check` + `eslint` + `build` 均通过。

## ContextBar 重构：去掉进度环，显示公共上下文 token 数
- `GroupChatContextBar.vue` 移除 `t-progress` 圆环，触发点改为一个 chip：显示「当前公共上下文的 token 估算」（格式 `xx.xk`/`x.xm`，<1000 显示原数字）+ 小字「上下文」；点击 `t-popup` 面板显示 token 数、剩余/过期状态、过期小时数、新上下文/压缩。
- **公共上下文定义** = 所有成员被问到时都会注入的同一份文本（摘要前缀 + buildTranscript 历史）。在 `GroupChatEngine` 抽出 `buildPublicContext(discussion, chat)`（与 runResponders 真正发送内容一致），并新增 `estimateTokens(text)`（CJK 约 1 token/字、其余约 1 token/4 字符的启发式估算）。组件需接收 `discussion` prop（`DiscussionChat.vue` 已补传）。
- 验证：`pnpm check` + `eslint` + `build` 通过；node 单测估算算法合理（中英混合/纯英文/空串）。
